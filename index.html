Curso FrontEnd Avanzado - GIT & Github

Listado completo de comandos GIT

Antes de comenzar tendremos que tener o crear un proyecto (HTML, o lo que sea) con sus archivos y carpetas.
Para iniciar GIT debemos pulsar sobre el botón derecho del ratón y seleccionar la opción "Git Bash Here" en la carpeta de nuestro proyecto (en el explorador de archivos).
Hecho esto, nos vamos al terminal GIT y escribimos el siguiente comando: "git init".
Esto creará una carpeta oculta llamada ".git" en la raíz de nuestro proyecto.
Ahora para comprobar que todo esta correcto, escribimos el comando "git status -s" y nos mostrará el estado de nuestro proyecto.
Veremos que en los archivos aparecen unos signos de interrogación. Qué significa esto? Que esos archivos no están siendo seguidos por GIT.
Para que GIT comience a seguir esos archivos, debemos añadirlos al "staging area" (área de preparación) con el comando "git add .".
Tecleamos el siguiente comando para añadir el archivo "index.html" al área de preparación: "git add index.html".
Ahora si escribimos el comando "git status -s" veremos que el archivo "index.html" ha pasado a la sección de "Changes to be committed". Veremos que ahora el nombre del archivo está precedido de la letra "A"
Ahora con el comando "git commit -m 'mensaje'" podemos confirmar los cambios realizados y enviarlo al repositorio. Entre comillas pondremos una breve descripción del archivo o de los cambios realizados sobre el mismo.
Ahora si escribimos el comando "git status -s" veremos que el archivo "index.html" ha desaparecido. Esto es porque en el terminal GIT sólo vemos los archivos que no han sido enviados al repositorio o bien han sido modificados de nuevo
Ahora si modificamos el archivo y escribimos el comando "git status -s" veremos que ahora el nombre del archivo está precedido de la letra "M" en color rojo
Ahora tendremos que volver a añadir el archivo al área de preparación con el comando "git add index.html". con "git status -s" veremos que ahora la "M" está en color verde. Ahora ya podremos enviarlo al repositorio. Podemos probar a saltarnos este paso y ver lo que pasa.
Restaurar archivo a la versión anterior: "git reset --hard número-copia".
Si por error nos hemos equivocado y queremos volver a lo que teníamos antes de hacer el último commit, podemos usar el comando "git reset número-copia". Si no recordamos el número de copia podemos verlo en el archivo ".git\logs\HEAD". En el veremos los textos identificativos de cada actualización y el número de código que se corresponderá con los siete primeros dígitos del número que aparece sobre el párrafo correspondiente:

8ccd9023643b369fe25967240166b1eb9f3f6a79 9a84bb0a092c82880c1d1a47ceefed2dcc2041cd RafaGuti 1746781006 +0200 commit: Archivo con Introducción GIT v3


Cualquier cambio realizado en el terminal de GIT se actualizará automáticamente en nuestro editor (Visual Studio Code, DreamWeaver, etc).

Resumen

  1. "git add nombre-archivo" para añadir un archivo al área de preparación.
  2. "git commit -m 'mensaje'" para enviar el archivo al repositorio.
  3. "git status -s" para ver el estado de los archivos.
  4. "git log --oneline" para ver el historial de cambios.
  5. Tras cualquier cambio en el fichero (index.html) tendremos que efectuar los pasos 1 y 2 para que GIT lo reconozca. Nota: el 'mensaje' tendremos que cambiarlo por otro, por ej.: 'actualización de estilos"
  6. Con "git add ." añadimos todos los archivos al área de preparación.
    ... más comandos de GIT ... <>Para poder modificar el mensaje del último commit, podemos usar el comando "git commit --amend -m 'nuevo mensaje'".
    Esto nos abrirá el Editor bin de git en nuestro editor (VSCode, DreamWeaver, etc) y podremos modificar el mensaje del último commit.
    Una vez modificado, guardamos el archivo y cerramos el editor.
    Si escribimos el comando "git log --oneline" veremos que el mensaje ha cambiado.
  • Ahora vamos a subir nuestro código a Hithub. Para ello vamos a la página principal de Hithub y nos registramos. Hecho esto, vamos a crear nuestro primer repositorio, pulsamos en "new" y le damos un nombre. El resto de opciones las dejamos por defecto.
    Una vez creado nos da una serie de opciones para subir nuestro código.
    En nuestro caso, como ya tenemos un documento creado, vamos a seleccionar la opción "git remote add origin https://github.com/rafaelgutierrezolivar/CursoGit.git" y a continuación "git push -u origin master".
    Esto subirá nuestro código a Hithub.
    Probablemente nos pide el usuario y la contraseña de Hithub.
    Una vez subido, si vamos a nuestro repositorio en Hithub veremos que ya tenemos nuestro código subido.
    Si queremos ver el historial de cambios, pulsamos en "commits" y veremos el historial de cambios realizados.
    Si queremos ver el código, pulsamos en "code" y veremos el código subido.
  • Si realizamos algún cambio en nuestro proyecto, para subirlo al repositorio Github tecleamos lo siguiente en la terminal de Git: "git push origin master". Si tenemos una sesión abierta en Github no funcionará correctamente.
  • También podemos editar directamente desde Github y que se actualice en local, es decir, en nuestro ordenador. Para ello seelccionamos el documento y pulsamos sobre el botón "de lapiz" para editar el documento. Hacemos los cambios oportunos y pulsamos sobre el botón "Commint changes". Si volvemos a nuestro documento en nuestro ordenador veremos los cambios automáticamente.
    Crear etiquetas y clonar repositorios.
  • Una vez que tengamos nuestro proyecto listo, y funcionando podemos crear una etiqueta del tipo nombre-versión y guardarlo todo. En la terminal de Git tecleamos lo siguiente: "git tag 12-05-25_v1 "nombre del proyecto". Pero para que se pueda ver en el repositorio Github tendremos que subirlo de la siguiente forma: "git push --tags".
    Para clonar un proyecto en caso de necesidad (borrado de datos, pérdida y rotura de ordenador, etc.) vamos a Github y en la pantalla principal de nuestro proyecto pulsamos sobre el botón "code". Aqui tenemos varias opciones, copiar la direccion del enlace, bajarla en zip, etc. En nuestro caso vamos a seleccionar la primera opción, para luego pegarla en el terminal de Git.
    Acto seguido en la terminal escribimos "git clone dirección-del-enlace".
    Ramas o Branch. Te permiten trabajar en paralelo sin afectar al proyecto principal
  • Para crear una rama tecleamos "git branch nombre-rama".
  • para ver las ramas y la rama actual tecleamos "git branch", y para cambiarnos de rama "git checkout nombre-rama
    Fusionar ramas
  • Para fusionar ramas, primero nos tenemos que cambiar a la rama principal (master) con el comando "git checkout master".
    Una vez en la rama principal, tecleamos "git merge nombre-rama" para fusionar la rama creada con la rama principal.
    Si queremos eliminar una rama, estando en la rama principal, tecleamos "git branch -d nombre-rama".
